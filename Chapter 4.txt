4.3 Таблицы SQL

Представление - Псевдо таблица, состоящая из столбцов в разных таблицах, и строк подходящих под заданные ограничения.
Курсор - Тоже самое, что и представление, только по строкам таблицы можно перемещаться с помощью команды fetch, внося изменения в каждую строчку.

4.4 Синтакчические конструкции SQL

-Предложения(Statement)
-Идентификаторы(имена)
-Константы
-Операторы
-Зарезервированные и ключевые слова
-Псевдостолбцы и таблица DUAL

Statement
Четких критериев по оформелнию нет.
Все предложения деляется на три группы:
	Манипулирование данными DML
	Определение данных DDL
	Оперделения доступа к данных DCL

Идентификаторы - имена таблиц, столбцов, курсоров, представлений. Бывают простные и с разделителем. Простые содержат буквы,цифры и знак подчеркивая, иногда набор символов другой, ну и не могут быть зарезервироваными словами. Идентификатор помещенный в кавычки "" симает ограничения на набор симолов и использование зарезервированных слов.

Константы
Числовые значения, строки символов,представления даты и времени, булевые значения. Которые не являются идентификаторыми или клчючевыми словами.
Числовые константы:
	11
	-11
	11.1
	+2
	5E6
	-9e-2
Допустимые символы в числовых константах: [0-9]+-$.Ee
Запятая разделитель элеметов. 5,6 будет интерпритироваться как 5 и 6

Булевые константы:
	TRUE
	FALSE

Константы даты:
	15-10-2017 15:06:45
	15-ОКТЯБРЬ-2019 11:45:20

Строковые константы заключаются в одинарные кавычки'', любой символ можно представить в виде строковой константы. Одинарный апостроф экранируется вторым 'Д''Артаньян' - соответсвует строве Д'Артаньян.

NULL
-Специальное значение
-Нельзя помещать с столбцы определенные как NOT NULL.
-NULL при сравнении с другим NULL вернет отрицательное значение.
-Для проверки на NULL необходимо испозовать спец. опертор IS NULL и IS NOT NULL.
-Столбец содержащий NULL значение игнорируется агрегатными функциями AVG,SUM,COUNT.
-Если столбцы с NULL перечислены в GROUP BY, то из всех данных с NULL в столбце будет выбрана только одна запись.
-ORDER BY и DISTINCT ставят столбцы с NULL в конец в произвольном порядке

Операторы
-Арифметические, все как везде +-/*
-Конкатенации || соединяют две строки
-Присваивания =, вроде для изменения названия столбца
-Сравнения = >< >=<=
Результат может быть TRUE,FALSE,UNKNOWN(Если сравнения между разными типами), NULL(Если хотя бы один из операндом NULL)
IS NULL, IS NOT NULL - возвращает TRUE или FALSE

Логические Операторы
AND,OR,NOT - самые обычные
ALL(Все),ANY(Хотя бы один),SOME(Больше одного или все) - агрегирующие, вернут TRUE, когда TRUE нужное кол-во операндов
BETWEEN - вернет TRUE если операнд находится внутри диапазона
EXIST - вернет TRUE если подзапрос вернет хотя бы одну строку 
IN - вернет TRUE если операнд равен одному из списка или строкам возращыемым подзапросом
LIKE - вернет TRUE если операнд совпадает с шаблоном

Унарные +- Делают число положительным или отрицательным. Интресно + -1 вернет 1 ?

Типы данных
	Символьные
		CHAR - символьные строки фиксированной длины
		VARCHAR - символьные строки переменной длины, пробелы по краям обрезаются
		NATIONAL CHARACTER - символьные данные связанные с назиональными символами
	Бинарные данные
		BLOB,CLOB - просто бинарные данные для хранения файлов
	Числовые
		NUMERIC(p,s) - числа с регулируемой точностью p и мастштабом s
	Дата/Время
		DATE
	Связь с данными
		DATALINK - определяют ссылку на файл или другой ресурс за пределами базы данных
	Интрервальные
		INTERVAL - определяет набор временых значений или промежуток времени
	XML - Данные формата xml
